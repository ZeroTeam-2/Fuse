# Design — выходы из примера ответа

## Context

`OpenApiParserService.extractOutputs` берёт первый content-объект успешного ответа и отдаёт `extractSchemaFields(schema)`; при `schema: {}` полей нет. Пример ответа может жить в трёх местах OpenAPI: `content[mime].example` (наш живой случай), `schema.example`, `content[mime].examples` (карта именованных примеров с `value`). Сейчас `getFirstContentSchema` возвращает только схему — примеры content-уровня теряются.

## Goals / Non-Goals

**Goals:** выходы из примера, когда разбор схемы дал пусто; признак коллекции и поэлементная схема массивов — теми же правилами, что у схемного пути; поведение при настоящей схеме не меняется.

**Non-Goals:** фолбэк для входов (`requestBody` без схемы — не наш случай, у него пример другой формы `@filename`); слияние схемы и примера (пример только «затыкает» полное отсутствие полей); глубина больше одного уровня вложенности.

## Decisions

- **D1. Триггер фолбэка — пустой результат разбора схемы**, а не отсутствие схемы: `schema: {}` и `schema: {type: object}` без properties одинаково бесполезны. Схема с хотя бы одним полем полностью выигрывает — примеры не подмешиваются.
- **D2. Источник примера по приоритету**: `content.example` → `schema.example` → первый `content.examples[*].value`. Первый непустой объект/массив побеждает.
- **D3. Вывод полей из значения примера** (`fieldsFromExampleValue`): объект → поля по ключам, `type` по JS-типу (`string/number/boolean/array/object`), значение в `ex`, `required` не ставим (пример обязательность не сообщает); массив → `isArray: true` + поля первого объектного элемента; поле-массив объектов внутри объекта → `items` из первого элемента (один уровень, как у схемы). Массив скаляров — тип `array` без `items`, симметрично схемному пути.
- **D4. Место в коде**: расширить `getFirstContentSchema` до `getFirstContent` (возвращает content-объект целиком), фолбэк — внутри `extractOutputs`. Входы продолжают пользоваться схемой как раньше.

## Risks / Trade-offs

- [Пример может врать о типах (`example: {status: "string"}` — плейсхолдеры)] → типы выйдут строками; это ровно то, что написал автор спеки, и лучше, чем отсутствие полей. Резолвер рантайма типами выходов не пользуется.
- [Именованные `examples` могут описывать разные формы ответа] → берём первый попавшийся объект/массив; спорные спеки чинить переимпортом после правки — не наша задача.

## Migration Plan

Кода миграции нет. Существующие приложения обновляются кнопкой переимпорта спеки.
