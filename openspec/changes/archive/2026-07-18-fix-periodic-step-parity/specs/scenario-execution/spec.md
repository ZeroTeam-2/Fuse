# scenario-execution — delta

## MODIFIED Requirements

### Requirement: Подстановка входов по месту в HTTP-запросе

Разрешённые входы шага MUST попадать в запрос согласно **месту** каждого поля (`SchemaField.loc`): path-параметры подставляются в шаблон пути, query — в строку запроса, header — в заголовки, остальные — в тело. Плейсхолдер в шаблоне пути (`{guid}`) MUST иметь приоритет над схемой: если поле названо в пути, его значение обязано уехать в путь, даже если схема эндпоинта разошлась с реальностью после переимпорта. Значение path-параметра MUST быть URL-экранировано. Поле, которого нет в схеме эндпоинта, MUST попадать в тело запроса.

Требование распространяется на **все** шаги, вызывающие endpoint, включая «Периодический запрос»: worker MUST разрешать схему endpoint periodic-шага (по `endpointId`, для легаси-шагов — по совпадению метода и пути опроса среди endpoints приложения) и выполнять **каждую итерацию опроса** с полным набором входов — path-параметры в пути, query в строке запроса, header-входы в заголовках, body-входы в теле (для методов с телом). Входы разрешаются один раз перед началом опроса и не меняются между итерациями.

Шаг MUST NOT уходить в вызов с неподставленным плейсхолдером в URL.

#### Scenario: Path-параметр подставляется в URL

- **WHEN** шаг вызывает `GET /collections/{guid}`, а вход `guid` разрешён в `6b7e9beb-1968-43c9-9e4f-1cd417fd4094`
- **THEN** worker вызывает `{baseUrl}/collections/6b7e9beb-1968-43c9-9e4f-1cd417fd4094`, а не путь с литеральным `{guid}`

#### Scenario: Входы расходятся по местам

- **WHEN** у шага есть входы с `loc` `path`, `query`, `header` и `body`
- **THEN** каждый уезжает в своё место запроса: путь, строку запроса, заголовки и тело соответственно

#### Scenario: Периодический опрос уходит с полным набором входов

- **WHEN** periodic-шаг опрашивает endpoint с path-параметром, query-параметром и header-входом (например, заголовком авторизации)
- **THEN** каждая итерация опроса вызывает URL с подставленным path-параметром и query-строкой и передаёт header-вход в заголовках запроса

#### Scenario: Body-вход периодического POST-опроса

- **WHEN** periodic-шаг опрашивает endpoint методом POST, и у него есть разрешённые body-входы
- **THEN** каждая итерация опроса отправляет эти входы JSON-телом запроса

#### Scenario: Легаси periodic-шаг без схемы не регрессирует

- **WHEN** periodic-шаг создан до появления `endpointId`, и endpoint с совпадающим методом и путём у приложения не найден
- **THEN** worker исполняет опрос как раньше — плейсхолдеры пути заполняются из разрешённых входов, остальные входы уходят в тело согласно правилу «поле вне схемы попадает в тело»
