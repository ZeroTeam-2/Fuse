# Design: merge-file-step-into-api-step

## Context

После `implement-file-step-upload` (архив 2026-07-17) весь файловый конвейер работает: dropzone грузит в MinIO до сабмита, в `page:submit` едет `UploadedFileRef`, worker читает объект и шлёт провайдеру multipart, парсер помечает `format: binary` как `type: "file"`. Но всё это висит на типе шага `file`, у которого нет конфигуратора в билдере — рабочий шаг существует только благодаря ручной правке БД. Api-шаг при этом умеет всё, кроме multipart-кодирования тела: живой прогон llm-doc-recognizer падал на 422 ровно из-за JSON-тела.

## Goals / Non-Goals

**Goals:**

- Один универсальный шаг вызова endpoint'а: JSON или multipart — по наличию файлового входа, автоматически.
- «Файл» исчезает из палитры; существующие file-шаги дают понятную доменную ошибку.
- Стадия обработки — композиция api + periodic (уже настраиваемая из UI).
- Терминальные статусы ошибки в цикле опроса.

**Non-Goals:**

- Изменения загрузки на странице (PageRunner/`useFileUpload`) — не трогаем.
- Физическое удаление `FileStep` из типов и `executeFileStep`-ветки — только депрекация; вычистка после того, как в проде не останется старых документов.
- Несколько файлов в одном запросе — по-прежнему один файл на шаг (первая файловая ссылка).

## Decisions

### 1. Детекция multipart — по значению входа, а не по типу шага

`executeApiStep` после `resolveMappings` ищет среди разрешённых входов `UploadedFileRef` (type guard из shared). Нашёлся — тело собирается как `FormData` (файл из MinIO + текстовые body-входы), нет — прежний JSON-путь без изменений. Альтернатива «по схеме endpoint'а (`type: "file"`)» отвергнута как единственный критерий: легаси-схемы хранят файловое поле строкой; значение-ссылка — самый надёжный сигнал, схема же уточняет **имя** поля (приоритет: body-вход `type: "file"` → ключ привязки → единственный незанятый body-вход → `file`) — логика из прошлого change переносится как есть.

### 2. `executeFileStep` → доменная ошибка, `FileStep` → `@deprecated`

Старые file-шаги в проде — нерабочие заглушки (до этой недели исполнение возвращало placeholder), поэтому ломать нечего: ветка `case "file"` заменяется `StepExecutionError` с текстом «тип шага устарел — используйте api-шаг с файловым входом». Тип остаётся в union (старые документы читаются билдером), помеченный `@deprecated`. Вычистка типов — отдельной уборкой позже.

### 3. Обработка после загрузки — periodic-шаг, без спецмеханизмов

`statusEndpoint` файлового шага дублировал периодический запрос. Вместо него — обычная композиция: api-шаг возвращает ответ провайдера (`task_id`), periodic-шаг маппит `task_id: s0:task_id` в `pollPath` `/task/{task_id}`. Подстановка выходов в путь и прогресс уже работают. Хелпер `pollUntilComplete` остаётся общим.

### 4. Терминальные статусы ошибки опроса

`isPollComplete` дополняется парой `isPollFailed`: `status: "error" | "failed"` в ответе опроса → `StepExecutionError` с телом ответа (llm-doc-recognizer отвечает `{"status": "error", "error": ...}` на битый файл — сейчас это крутится до 5-минутного таймаута).

### 5. Миграция «Распознать документ»

Локальный сценарий: шаг 0 возвращается в `type: "api"` (`method`/`path`/`endpointId` восстанавливаются), добавляется шаг 1 `periodic` (`pollPath: "/task/{task_id}"`, `mappings: {task_id: "s0:task_id"}`, интервал 3 с). Правка БД скриптом, как и прежняя конвертация.

## Risks / Trade-offs

- **[Файловая ссылка в шаге без файлового endpoint'а]** → multipart уедет туда, где ждали JSON; но ссылка попадает во входы только через явную привязку dropzone — осознанное действие редактора. Приемлемо.
- **[Старые file-шаги в чужих сценариях]** → доменная ошибка с подсказкой вместо тихого placeholder — строго лучше прежнего поведения.
- **[`isPollFailed` по конвенции имён статусов]** → как и `isPollComplete`, эвристика (`error`/`failed`); ложных срабатываний на нормальных ответах не ожидается, ложные пропуски оставляют прежнее поведение (таймаут).

## Migration Plan

Код аддитивен к рантайму: деплой обычный. Скрипт миграции локального сценария выполняется вручную при применении change. Откат — revert; мигрированный сценарий работает и на старом коде (api + periodic — базовые типы).
