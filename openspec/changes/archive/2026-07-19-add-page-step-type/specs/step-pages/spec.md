# step-pages — delta

## ADDED Requirements

### Requirement: Шаг типа «Страница»
Страница SHALL существовать в сценарии как самостоятельный шаг типа `page`, а не как настройка другого шага. Шаг-страница SHALL нести ровно одну раскладку (`StepPage`) как своё основное содержимое; шаг-страница без единого блока MUST отклоняться валидацией при сохранении сценария. Раскладка SHALL открываться в том же полноэкранном визуальном конструкторе по кнопке «Открыть редактор» из панели настройки шага. Панель настройки шага-страницы SHALL показывать название шага и число размещённых элементов.

#### Scenario: Открытие конструктора у шага-страницы
- **WHEN** редактор сценария открывает панель настройки шага «Страница» и нажимает «Открыть редактор»
- **THEN** открывается полноэкранный конструктор страницы с палитрой элементов, листом страницы и инспектором свойств

#### Scenario: Пустая страница отклоняется
- **WHEN** сценарий сохраняется с шагом-страницей без единого блока
- **THEN** валидация возвращает ошибку, и сценарий не сохраняется

### Requirement: Ключ выхода блока ввода
Блок категории Ввод на шаге-странице SHALL объявлять **выход шага-страницы**: инспектор SHALL предлагать текстовое поле «Ключ выхода» (латиница, цифры, подчёркивание), под которым введённое пользователем значение станет полем результата шага. Ключи выходов MUST быть уникальны в пределах страницы; блок без ключа отдаёт значение под собственным идентификатором. Тип выхода SHALL определяться типом блока: `dropzone` — файл, остальные блоки ввода — строка. Выходы шага-страницы SHALL быть доступны последующим шагам как обычные выходы шага (маппинг «Из шага»).

#### Scenario: Значение блока уходит в выход шага
- **WHEN** редактор задаёт блоку ввода ключ выхода `inn`, а у следующего шага маппит вход на «Из шага → Страница → inn»
- **THEN** при запуске значение, введённое пользователем на странице, подставляется во вход следующего шага

#### Scenario: Дубликат ключа отклоняется
- **WHEN** редактор вводит ключ выхода, уже занятый другим блоком этой страницы
- **THEN** инспектор показывает ошибку, и ключ не сохраняется

### Requirement: Страница есть только у шага «Страница»
Раскладка страницы SHALL существовать только в составе шага типа `page`: поле встроенной страницы у шагов других типов MUST быть удалено из модели, а панели их настройки MUST NOT содержать секцию «Страница». Старые документы со встроенными страницами не поддерживаются — локальные данные сбрасываются и пересеиваются.

#### Scenario: У обычных шагов нет секции «Страница»
- **WHEN** редактор открывает панель настройки шага типа `api`, `scenario`, `delay` или `periodic`
- **THEN** секции «Страница» в панели нет

## REMOVED Requirements

### Requirement: Привязка страницы к шагу
**Reason**: Страница перестаёт быть настройкой произвольного шага — она становится самостоятельным шагом типа `page` (см. ADDED «Шаг типа „Страница“»).
**Migration**: Без миграции: поле встроенной страницы удаляется из модели, локальная база сбрасывается (`pnpm reset`) и пересеивается сидом с page-шагами.

### Requirement: Привязка блока ввода к входному параметру шага
**Reason**: У шага-страницы нет «входных параметров шага-носителя»; блок ввода теперь объявляет ключ выхода шага-страницы (см. ADDED «Ключ выхода блока ввода»), а следующие шаги забирают значение штатным маппингом «Из шага».
**Migration**: Без миграции: старая семантика привязки удаляется вместе со встроенными страницами, данные пересеиваются.

## MODIFIED Requirements

### Requirement: Сохранение страницы
Система SHALL сохранять раскладку страницы в составе шага-страницы сценария. После «Готово» в конструкторе панель настройки шага-страницы SHALL отражать число размещённых элементов. При запуске сценария страницы MUST отображаться пользователю в порядке исполнения их шагов: блоки ввода собирают значения в результат шага-страницы, блоки отображения показывают данные пройденных шагов.

#### Scenario: Сохранение страницы
- **WHEN** редактор нажимает «Готово» в конструкторе
- **THEN** раскладка сохраняется в шаге-странице, и панель настройки шага показывает число элементов страницы
