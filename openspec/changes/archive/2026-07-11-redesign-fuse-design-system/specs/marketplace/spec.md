## MODIFIED Requirements

### Requirement: Страница карточки
Карточка сценария SHALL иметь вкладки: «Обзор» (описание в rich-формате, число запусков, число поставщиков API, число endpoints, категория, кнопка «Перейти к запуску»), «Запуск» и «Сервисы и endpoints» (список поставщиков API и задействованных endpoints с методами и путями).

Страница карточки SHALL быть публичной: гость видит её целиком, включая вкладку «Запуск» с превью шагов сценария. Публичное представление опубликованного сценария MUST включать его шаги. Сам запуск MUST требовать аутентификации (см. capability `auth-profile`).

#### Scenario: Просмотр обзора
- **WHEN** пользователь открывает карточку из каталога
- **THEN** система показывает вкладку «Обзор» с отрендеренным описанием и метриками карточки

#### Scenario: Гость смотрит вкладку «Запуск»
- **WHEN** неаутентифицированный пользователь открывает вкладку «Запуск» карточки
- **THEN** система показывает панель запуска с превью шагов сценария (не сообщение об ошибке доступа), но запуск остаётся недоступным до входа
