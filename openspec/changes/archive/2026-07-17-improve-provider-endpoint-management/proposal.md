## Why

Приложения-провайдеры с большим числом endpoints сейчас неудобны в управлении: endpoints выводятся плоским списком без смысловой группировки, широкие описания из OpenAPI ломают вёрстку превью импорта, общий компонент пагинации бесконечно растягивается вместо переключения страниц, а у провайдера нет способа управлять окружениями (Base URL для prod/stage/dev). Всё это усложняет и импорт больших спецификаций, и сборку сценариев.

## What Changes

- При разборе OpenAPI система извлекает **теги** операций и группирует endpoints по смысловым блокам (по первому тегу; endpoints без тегов — в блок «Прочее»).
- Карточка провайдера показывает endpoints **сворачиваемыми блоками** (collapsible) с **сквозным поиском** по методу, пути, summary и имени блока/тега.
- Превью импорта OpenAPI показывает endpoints теми же сворачиваемыми блоками с **корректной вёрсткой** — длинное описание усекается/переносится и не выходит за пределы контейнера. Перечислять все endpoints для создания/обновления не обязательно.
- При создании шага (выбор endpoint) список endpoints тоже **группируется в сворачиваемые блоки** со **сквозным поиском в едином стиле** с карточкой провайдера.
- Общий компонент `Pagination` чинится: вместо чипа на каждую страницу — оконный вид с многоточиями («1 … 4 5 6 … 20»), корректно переключающий/скипающий страницы. Фикс распространяется на все места использования.
- У провайдера появляется управление **Environments**: по умолчанию есть **Prod**, можно создавать и удалять окружения. В каждом окружении хранится **набор переменных**, в котором пока один элемент — **`baseUrl`** (модель расширяема под будущие переменные). Исполнение резолвит путь относительно `baseUrl` выбранного окружения, сохраняя обратную совместимость с текущим `App.baseUrl`.
- В редакторе сценария появляется отдельная вкладка **«Окружения»**: для каждого поставщика (app), задействованного в шагах сценария, выбирается используемое окружение (по умолчанию Prod). Выбор сохраняется в сценарии, и исполнение каждого шага резолвит `baseUrl` из окружения, выбранного для его поставщика.

## Capabilities

### New Capabilities
- `api-app-environments`: окружения провайдера (по умолчанию Prod, создание/удаление, набор переменных с базовым `baseUrl`) и их использование при исполнении шагов.
- `pagination-control`: поведение общего компонента пагинации — оконный вид с многоточиями и корректное переключение страниц.

### Modified Capabilities
- `api-app-management`: извлечение тегов OpenAPI и группировка endpoints по сворачиваемым блокам; сквозной поиск по endpoints на карточке провайдера; корректная вёрстка превью импорта при широких описаниях; связь базового URL с окружениями (Prod seed из импортированного `baseUrl`).
- `scenario-builder`: выбор endpoint при создании шага — группировка по сворачиваемым блокам и сквозной поиск в едином стиле; отдельная вкладка «Окружения» с выбором окружения для каждого поставщика сценария.

## Impact

- **Backend** (`apps/backend/src/apps/`): `openapi-parser.ts` (извлечение `tags`), `app.schema.ts` (поле `tag` у endpoint, сущность `environments` с набором переменных у приложения), `apps.controller.ts`/`apps.service.ts` (CRUD окружений, резолв `baseUrl` при исполнении), `execution/worker.service.ts` (использование `baseUrl` окружения, выбранного для поставщика в сценарии), `scenarios/` схема/сервис (хранение выбора окружений по поставщикам), миграция/бэкфилл окружения Prod из `baseUrl`.
- **Shared** (`packages/shared/src/types`): типы `Endpoint.tag`, `Environment` (набор переменных), выбор окружений сценария, обновление `App`, `ImportPreviewResult`, `Scenario`.
- **Frontend** (`apps/frontend`): `components/ui/Pagination.vue` (оконный вид), новый компонент группируемого списка endpoints с поиском (переиспользуется в карточке провайдера, превью импорта и StepPicker), `pages/my/apps/[id]/index.vue`, `pages/my/apps/new.vue`, `pages/my/apps/[id]/update.vue`, `components/scenario/StepPicker.vue`, `components/ui/EndpointRow.vue` (усечение описания), UI управления Environments на карточке провайдера, вкладка «Окружения» в редакторе сценария (`pages/my/scenarios/[id]/edit.vue` + `stores/scenario-editor.ts`).
- Обратная совместимость: существующие приложения без окружений получают окружение Prod с текущим `baseUrl`; сценарии без выбора окружений используют Prod для каждого поставщика; endpoints без тегов попадают в блок «Прочее».
