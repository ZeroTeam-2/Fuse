## Context

В Fuse уже реализовано удаление приложений: на странице `apps/frontend/pages/my/apps/[id]/index.vue` есть danger-кнопка «Удалить» и модальное окно подтверждения (`Modal` + `Button variant="danger"`), после подтверждения вызывается `DELETE /api/apps/{id}` и выполняется редирект на список. Бэкенд сценариев уже имеет зеркальный эндпоинт `ScenariosController_delete` (`DELETE /api/scenarios/:id`), но в редакторе сценария `apps/frontend/pages/my/scenarios/[id]/edit.vue` кнопки удаления нет. Изменение чисто фронтендовое и сводится к тиражированию устоявшего паттерна.

## Goals / Non-Goals

**Goals:**
- Дать пользователю возможность удалить сценарий прямо из редактора.
- Визуально и поведенчески повторить паттерн удаления приложения (danger-кнопка с иконкой + модальное подтверждение).
- Переиспользовать существующий бэкенд-эндпоинт без изменений API.

**Non-Goals:**
- Пакетное удаление сценариев из списка.
- Мягкое удаление / корзина / восстановление.
- Изменения бэкенда или схемы данных.

## Decisions

### Решение 1: Кнопка в шапке рядом с PublishButton
Кнопка «Удалить» размещается в строке заголовка редактора (`div.flex.items-center.gap-4`), сразу после `PublishButton` — симметрично расположению на странице приложения. Альтернатива (вынести в отдельное меню/вкладку «Опасные действия») отвергнута: нарушает консистентность с приложениями и удлиняет путь к действию.

### Решение 2: Обязательное модальное подтверждение
Удаление необратимо, поэтому нужен `Modal` с предупреждением и кнопками «Отмена» / «Удалить». Текст предупреждения адаптирован под сценарий (удаляются шаги и публикации в маркетплейсе). Повторяет структуру модалки удаления приложения 1:1.

### Решение 3: На успех — редирект на /my/scenarios
После успешного `DELETE` вызывается `navigateTo("/my/scenarios")`, как и при удалении приложения (`navigateTo("/my/apps")`). На ошибке модалка закрывается (поведение `deleteApp`), состояние ошибки не накапливается.

### Решение 4: Переиспользование существующих UI-примитивов
`Button` с `variant="danger"`, слот `#left` с `Icon name="trash-2"`, компонент `Modal` — всё уже есть. Новых зависимостей не появляется.

## Risks / Trade-offs

- [Жёсткое удаление без восстановления] → Осознанный выбор, соответствует текущей модели приложений. Модальное подтверждение снижает риск случайного удаления. Сценарии демо-пользователя пересоздаются при сидировании, поэтому для dev-среды необратимость некритична.
- [Отсутствие обработки ошибок на UI] → Поведение повторяет `deleteApp` (модалка закрывается, без тоста). При желании позже можно добавить единый механизм ошибок для обоих экранов — за рамками данного изменения.
