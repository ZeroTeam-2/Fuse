# scenario-execution — delta

## MODIFIED Requirements

### Requirement: Отображение страниц шагов при запуске
При исполнении сценария система SHALL показывать страницы как раскладку из блоков (строки × блоки на сетке из 6 колонок). Панель запуска SHALL рендерить раскладку обобщённо: блоки ввода (`input`, `select`, `dropzone`, `richtext`) собирают данные (обязательные блокируют продолжение), блоки отображения (`paragraph`) показывают текст или подставленное значение из результата пройденного шага. Перед показом страницы движок SHALL разрешить из результатов предыдущих шагов и доставить в payload события `page:required` по `blockId`: значения блоков отображения (массивный выход источника сводится к одному значению теми же правилами, что и резолв маппингов) и варианты блоков `select` с `optionsSource` (массивное поле/результат шага-источника разворачивается в список строк-вариантов).

Исполнение шага типа `page` зависит от состава блоков:
- **Есть блоки ввода** — `Run` переходит в статус `waiting_input`, публикуется `page:required`; данные формы отправляются через WebSocket (`page:submit`) или POST-запрос. Собранные значения MUST записываться **результатом шага-страницы**: объект «ключ выхода блока (либо его идентификатор) → значение»; обязательные ключи MUST быть непустыми. После записи результата worker продолжает исполнение; последующие шаги читают значения страницы штатным резолвом `s{idx}:{key}` (включая ссылку на загруженный файл `UploadedFileRef` из dropzone-блока — она доходит до api-шага как файловый вход и включает multipart-режим).
- **Только блоки отображения** — worker публикует `page:required` с резолвнутыми данными, записывает результатом шага пустой объект и продолжает исполнение **без ожидания** сабмита. Если такой шаг — последний, панель запуска MUST оставить страницу видимой как финальный экран после завершения запуска (событие `run:done` не скрывает её).

Значения, собираемые шагом-страницей, MUST NOT запрашиваться общей формой запуска: они не порождают дескрипторов ручного ввода.

Шаги других типов со встроенной страницей (легаси `BaseStep.page`, созданные до изменения) MUST исполняться по прежней семантике: пауза перед исполнением шага-носителя, подстановка данных страницы во входы шага по привязкам блоков, покрытые страницей значения не запрашиваются формой запуска.

#### Scenario: Страница ввода посреди потока
- **WHEN** исполнение доходит до шага «Страница» с блоками ввода
- **THEN** поток приостанавливается (`waiting_input`), пользователь заполняет блоки, после отправки значения записываются результатом шага-страницы, и исполнение продолжается

#### Scenario: Следующий шаг читает значение страницы
- **WHEN** вход api-шага замаплен на выход `inn` предшествующего шага-страницы, и пользователь отправил страницу со значением `7707083893`
- **THEN** worker разрешает вход из результата шага-страницы и вызывает endpoint со значением `7707083893`

#### Scenario: Финальная страница с текстом ответа
- **WHEN** последний шаг сценария — «Страница» только с блоками отображения, привязанными к выходам предыдущих шагов
- **THEN** worker публикует страницу с подставленными значениями, завершает запуск без ожидания сабмита, и панель запуска оставляет страницу видимой как экран результата

#### Scenario: Display-only страница не блокирует поток
- **WHEN** исполнение доходит до шага «Страница» без блоков ввода, за которым следуют другие шаги
- **THEN** страница публикуется для показа, результатом шага пишется пустой объект, и worker немедленно продолжает следующий шаг

#### Scenario: Файл со страницы уходит в api-шаг
- **WHEN** dropzone-блок шага-страницы с ключом выхода `doc` загрузил файл, а файловый вход следующего api-шага замаплен на `s{idx}:doc`
- **THEN** worker находит `UploadedFileRef` в результате шага-страницы и отправляет api-запрос телом `multipart/form-data` с этим файлом

#### Scenario: Блок отображения показывает данные пройденного шага
- **WHEN** страница шага содержит блок отображения, привязанный к выходу предыдущего шага, и исполнение доходит до этого шага
- **THEN** движок подставляет значение из результата шага-источника в payload `page:required`, и панель запуска показывает это значение в блоке

#### Scenario: Источник данных отображения недоступен
- **WHEN** блок отображения привязан к шагу или полю, которого нет в результатах
- **THEN** блок показывается пустым, а исполнение шага не прерывается ошибкой

#### Scenario: Варианты select из массива пройденного шага
- **WHEN** страница шага содержит блок `select` с `optionsSource`, указывающим на массивное поле выхода предыдущего шага, и исполнение доходит до этого шага
- **THEN** движок разворачивает значения массива в список вариантов в payload `page:required`, и панель запуска предлагает пользователю выбор из этих вариантов

#### Scenario: Легаси-шаг со встроенной страницей
- **WHEN** worker исполняет шаг типа `api` со встроенной страницей, созданной до изменения
- **THEN** запуск приостанавливается перед исполнением шага, данные страницы подставляются во входы шага по привязкам блоков, и шаг исполняется как раньше
