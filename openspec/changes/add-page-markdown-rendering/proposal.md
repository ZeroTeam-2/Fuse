# Блок «Текст»: рендеринг Markdown и переименование «Абзаца»

## Why

Блок «Абзац» (`paragraph`) на страницах шагов показывает текст как plain text (`whitespace-pre-wrap`). Когда блок привязан к данным пройденного шага, из API часто приходит текст в формате Markdown — нередко с экранированными переводами строк (`\n` как литеральная последовательность). Сейчас пользователь видит «сырой» текст со звёздочками, решётками и `\n` вместо отформатированного содержимого.

## What Changes

- Блок `paragraph` переименовывается в палитре конструктора из «Абзац» в «Текст» — с настройкой формата он больше, чем абзац. Тип в модели остаётся `paragraph`, миграция не нужна.
- У блока появляется настройка формата в инспекторе конструктора: «Обычный текст» (по умолчанию) или «Markdown». Модель блока получает опциональное поле `format`.
- В режиме Markdown блок на странице исполнения (PageRunner: панель запуска, playground, финальный display-only экран) рендерит своё содержимое как Markdown — и статический текст блока, и значение, разрешённое из результата пройденного шага. В режиме «Обычный текст» поведение прежнее (plain text, `whitespace-pre-wrap`).
- Строки из результатов API нормализуются перед рендером: литеральные экранированные последовательности (`\n`, `\t`, `\"`, `\\`) превращаются в реальные символы.
- Полученный из Markdown HTML проходит через существующий allowlist-санитайзер (`sanitizeRichText`) перед вставкой через `v-html` — та же защита, что у описаний маркетплейса.
- Санитайзер расширяется поддержкой безопасных ссылок (`<a>` c `href` только `http/https/mailto`, принудительные `rel="noopener noreferrer"` и `target="_blank"`).
- В конструкторе страницы редактирование абзаца остаётся «сырым» (contenteditable с исходным Markdown) — рендер применяется только на странице исполнения.

## Capabilities

### New Capabilities

_нет_

### Modified Capabilities

- `step-pages`: блок `paragraph` называется в конструкторе «Текст» и получает настройку формата (обычный текст / Markdown); в режиме Markdown содержимое при исполнении рендерится как Markdown (с нормализацией экранированных последовательностей и санитизацией HTML).

## Impact

- **Frontend**: `apps/frontend/components/run/PageRunner.vue` (ветка рендера `paragraph`), `apps/frontend/components/scenario/PageEditor.vue` (переключатель формата в инспекторе), `apps/frontend/utils/sanitizeRichText.ts` (поддержка `<a href>`), новая утилита рендера Markdown; новая зависимость — Markdown-парсер (`marked`).
- **Shared**: опциональное поле `format?: "text" | "markdown"` в `PageBlock` (`packages/shared/src/types`). Серверный `page-validator` permissive к неизвестным полям — правок бэкенда не требуется; старые страницы без поля читаются как «обычный текст».
- **Безопасность**: содержимое приходит из результатов внешних API и от авторов сценариев — весь HTML обязан проходить санитайзер; сырые HTML-теги внутри Markdown не пропускаются.
