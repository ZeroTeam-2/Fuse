# step-pages — delta: блок «Текст», настройка формата и рендеринг Markdown

## ADDED Requirements

### Requirement: Имя блока отображения — «Текст»
Блок типа `paragraph` SHALL называться в палитре конструктора «Текст». Тип блока в модели данных MUST остаться `paragraph` — сохранённые страницы не мигрируются.

#### Scenario: Палитра показывает блок «Текст»
- **WHEN** редактор открывает конструктор страницы
- **THEN** в разделе «Отображение» палитры блок называется «Текст», а сохраняется по-прежнему с типом `paragraph`

### Requirement: Настройка формата блока отображения
Блок `paragraph` SHALL иметь настройку формата содержимого: «Обычный текст» (значение по умолчанию) или «Markdown». Инспектор свойств MUST показывать переключатель формата только для блока `paragraph`. Выбор SHALL сохраняться в модели блока опциональным полем `format` (`"text" | "markdown"`); отсутствие поля или неизвестное значение MUST трактоваться как «Обычный текст» — существующие страницы рендерятся как раньше.

#### Scenario: Автор включает Markdown у блока «Текст»
- **WHEN** редактор выбирает блок «Текст» и переключает формат на «Markdown»
- **THEN** выбор сохраняется в модели блока и переживает переоткрытие конструктора

#### Scenario: Старые страницы не меняют вид
- **WHEN** открывается страница исполнения с блоком `paragraph`, сохранённым до появления настройки (без поля `format`)
- **THEN** блок рендерится как обычный текст — как до изменения

### Requirement: Рендеринг Markdown в блоке отображения
Блок `paragraph` с форматом «Markdown» на странице исполнения SHALL рендерить своё содержимое как Markdown: и статический текст блока, и значение, разрешённое из результата пройденного шага. Перед парсингом строка MUST нормализоваться — литеральные экранированные последовательности (`\n`, `\t`, `\"`, `\\`) заменяются на реальные символы; одиночный перевод строки SHALL рендериться как разрыв строки. Полученный HTML MUST проходить allowlist-санитизацию перед вставкой в DOM: разрешены только теги форматирования (абзацы, заголовки, списки, код, цитаты, выделение) и ссылки; сырые HTML-теги из исходного текста MUST NOT попадать в DOM как разметка. Блок с форматом «Обычный текст» MUST рендериться как plain text с сохранением переводов строк. В конструкторе страницы автор SHALL редактировать исходный Markdown-текст как раньше — рендер применяется только на странице исполнения.

#### Scenario: Markdown из результата API отображается отформатированным
- **WHEN** блок отображения с форматом «Markdown» привязан к полю результата пройденного шага, и API вернул строку `"## Итог\n**Готово**"` с литеральным `\n`
- **THEN** блок показывает заголовок «Итог» и жирное «Готово» — с реальным переводом строки, без звёздочек и решёток

#### Scenario: Статический текст блока рендерится как Markdown
- **WHEN** автор задал блоку `paragraph` формат «Markdown» и текст со списком `- пункт` и выделением `**важно**`
- **THEN** на странице исполнения текст отображается как маркированный список и жирное начертание

#### Scenario: Обычный текст не интерпретируется
- **WHEN** блок с форматом «Обычный текст» содержит `*текст*` и `# заголовок`
- **THEN** символы отображаются буквально, без курсива и заголовков

#### Scenario: Опасная разметка не попадает в DOM
- **WHEN** результат API, привязанный к Markdown-блоку, содержит `<script>`, inline-обработчики событий или ссылку со схемой `javascript:`
- **THEN** скрипты и атрибуты-обработчики вырезаются, у ссылки отбрасывается `href`, и в DOM попадает только разрешённая разметка

#### Scenario: Ссылки безопасны
- **WHEN** содержимое Markdown-блока содержит ссылку с `https`-адресом
- **THEN** ссылка рендерится кликабельной, открывается в новой вкладке и несёт `rel="noopener noreferrer"`
