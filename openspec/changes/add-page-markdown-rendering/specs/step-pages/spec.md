# step-pages — delta: рендеринг Markdown в блоке «Абзац»

## ADDED Requirements

### Requirement: Рендеринг Markdown в блоке отображения
Блок `paragraph` на странице исполнения SHALL рендерить своё содержимое как Markdown: и статический текст блока, и значение, разрешённое из результата пройденного шага. Перед парсингом строка MUST нормализоваться — литеральные экранированные последовательности (`\n`, `\t`, `\"`, `\\`) заменяются на реальные символы; одиночный перевод строки SHALL рендериться как разрыв строки. Полученный HTML MUST проходить allowlist-санитизацию перед вставкой в DOM: разрешены только теги форматирования (абзацы, заголовки, списки, код, цитаты, выделение) и ссылки; сырые HTML-теги из исходного текста MUST NOT попадать в DOM как разметка. В конструкторе страницы автор SHALL редактировать исходный Markdown-текст как раньше — рендер применяется только на странице исполнения.

#### Scenario: Markdown из результата API отображается отформатированным
- **WHEN** блок отображения привязан к полю результата пройденного шага, и API вернул строку `"## Итог\n**Готово**"` с литеральным `\n`
- **THEN** блок показывает заголовок «Итог» и жирное «Готово» — с реальным переводом строки, без звёздочек и решёток

#### Scenario: Статический текст блока рендерится как Markdown
- **WHEN** автор задал блоку `paragraph` текст со списком `- пункт` и выделением `**важно**`
- **THEN** на странице исполнения текст отображается как маркированный список и жирное начертание

#### Scenario: Опасная разметка не попадает в DOM
- **WHEN** результат API содержит `<script>`, inline-обработчики событий или ссылку со схемой `javascript:`
- **THEN** скрипты и атрибуты-обработчики вырезаются, у ссылки отбрасывается `href`, и в DOM попадает только разрешённая разметка

#### Scenario: Ссылки безопасны
- **WHEN** содержимое блока содержит Markdown-ссылку с `https`-адресом
- **THEN** ссылка рендерится кликабельной, открывается в новой вкладке и несёт `rel="noopener noreferrer"`
