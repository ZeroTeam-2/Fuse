# observability

## ADDED Requirements

### Requirement: Продуктовая аналитика Yandex Metrica (опционально)
Система MAY интегрировать Yandex Metrica во фронтенде для сбора продуктовой аналитики (просмотры страниц, запуски сценариев, публикации). Интеграция SHALL включаться только при наличии конфигурационной переменной окружения (`NUXT_PUBLIC_YANDEX_METRIKA_ID`); без неё скрипт счётчика не загружается. Ключевые пользовательские действия (запуск сценария, успешный/ошибочный запуск, публикация) SHALL отправляться как цели/события Metrica.

#### Scenario: Включена по конфигу
- **WHEN** задана переменная `NUXT_PUBLIC_YANDEX_METRIKA_ID`
- **THEN** фронтенд инициализирует счётчик Metrica и отправляет продуктовые события

#### Scenario: Отключена по умолчанию
- **WHEN** переменная `NUXT_PUBLIC_YANDEX_METRIKA_ID` не задана
- **THEN** ни счётчик, ни какие-либо запросы к Metrica не выполняются

### Requirement: Сбор метрик и логов в Yandex Cloud Monitoring / monium (опционально)
Бэкенд MAY отправлять структурированные логи и метрики в Yandex Cloud Monitoring (monium). Интеграция SHALL включаться только при наличии облачных учётных данных/переменных (`YC_*`); без них бэкенд выводит структурированные логи в stdout без внешних отправок. Система SHALL собирать ключевые эксплуатационные метрики: длину очереди выполнения, время выполнения шага, количество запусков по статусам и частоту ошибок.

#### Scenario: Включена по конфигу
- **WHEN** заданы облачные учётные данные Yandex Cloud
- **THEN** бэкенд отправляет метрики и структурированные логи в Yandex Cloud Monitoring (monium)

#### Scenario: Фолбэк на stdout
- **WHEN** облачные учётные данные не заданы
- **THEN** бэкенд ведёт структурированные логи только в stdout без отправки во внешний сервис мониторинга
