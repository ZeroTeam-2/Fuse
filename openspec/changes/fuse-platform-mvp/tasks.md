# Fuse Platform MVP — Tasks (Milestones)

## 1. Milestone 0 — Фундамент монорепо

- [ ] 1.1 Инициализировать `apps/backend`: NestJS + Mongoose, конфиг (env), health-check, `@nestjs/swagger` со схемой на `/api/schema.json`
- [ ] 1.2 Инициализировать `apps/frontend`: Nuxt 3 + Pinia, базовый layout (шапка: Маркетплейс / Мои сценарии / Мои API / профиль), API-клиент
- [ ] 1.3 Наполнить `packages/shared`: типы домена (App, Endpoint, Scenario, Step-union, Page, Run), enum'ы типов шагов/страниц, справочник категорий с подкатегориями
- [ ] 1.4 docker-compose: MongoDB + MinIO; pnpm dev-скрипты, проверка `gen:types` end-to-end
- [ ] 1.5 CI-заготовка: lint (oxlint), typecheck, unit-тесты на PR

## 2. Milestone 1 — Аутентификация и профиль (`auth-profile`)

- [ ] 2.1 Backend: модуль auth — регистрация, вход, JWT access+refresh в httpOnly cookie, guards
- [ ] 2.2 Backend: модуль users — профиль (ФИО, email), смена пароля с проверкой текущего, загрузка/удаление аватара
- [ ] 2.3 Frontend: страницы входа/регистрации, guard роутов
- [ ] 2.4 Frontend: страница профиля — личные данные, смена пароля с ошибками валидации, аватар
- [ ] 2.5 Frontend: сводка «Мои API» и «Мои сценарии» в профиле (пагинация, статусы публикации)

## 3. Milestone 2 — Приложения и импорт OpenAPI (`api-app-management`)

- [ ] 3.1 Backend: модель App + снимок спецификации; парсер OpenAPI (dereference, извлечение endpoints: метод/путь/summary, схемы in/out с примерами)
- [ ] 3.2 Backend: импорт по URL с SSRF-защитой (https, запрет приватных IP, таймауты) и внятными ошибками валидации
- [ ] 3.3 Backend: CRUD приложений, повторный импорт с diff (новые/deprecated endpoints), toggle публикации
- [ ] 3.4 Frontend: список «Мои API» (пагинация) и карточка приложения (endpoints, синхронизация, счётчик сценариев)
- [ ] 3.5 Frontend: мастер «Новое приложение» — форма, импорт, предпросмотр найденных endpoints, создание
- [ ] 3.6 Frontend: экран «Обновление API» — reimport, diff с метками «НОВЫЙ», сохранение
- [ ] 3.7 Тесты: парсер на реальных спеках (petstore + 2-3 публичных API), diff-логика

## 4. Milestone 3 — Конструктор сценариев (`scenario-builder`)

- [ ] 4.1 Backend: модели Scenario/Step (дискриминированный union), CRUD, валидация публикации (≥1 шаг), guard циклов вложенных сценариев
- [ ] 4.2 Backend: выдача upstream-схем для маппинга (выходы шагов 0..i-1, входы endpoint'а с локациями path/query/header/body)
- [ ] 4.3 Frontend: создание сценария — детали, обложка (drag-and-drop), rich-text описание (Tiptap: B/I/U, H1-H3, code), категория с поиском
- [ ] 4.4 Frontend: редактор потока — список шагов, вставка в позицию, drag-and-drop порядок, редактирование/удаление, индикатор мультисервисности
- [ ] 4.5 Frontend: пикер шага — тип «Endpoint API» (выбор приложения с поиском → endpoint), название шага
- [ ] 4.6 Frontend: маппинг входов — источник (upstream/user/const) с автоподстановкой по ключу/метке, дефолтные заголовки (`Bearer {{access_token}}`), панель выходов с примерами
- [ ] 4.7 Frontend: типы шагов «Задержка» (пресеты) и «Другой сценарий» (входы→выходы, без циклов)
- [ ] 4.8 Frontend: типы шагов «Файл» (режимы single/multipart, Content-Type, опрос статуса: endpoint, интервал, поле прогресса) и «Периодический запрос»
- [ ] 4.9 Публикация/скрытие сценария; блокировка публикации без шагов с подсказкой

## 5. Milestone 4 — Страницы шагов (`step-pages`)

- [ ] 5.1 Shared/Backend: модель Page (fields/file/text) в составе Step, дефолтные шаблоны типов
- [ ] 5.2 Frontend: редактор страницы — переключение типов с сохранением заголовка, поля (метка/placeholder/обязательное, добавление/удаление), тексты кнопок, форматы и лимит МБ, текстовое тело
- [ ] 5.3 Frontend: live-превью «Как увидит пользователь»
- [ ] 5.4 Frontend: бейдж страницы на шаге в редакторе потока, удаление страницы

## 6. Milestone 5 — Маркетплейс (`marketplace`)

- [ ] 6.1 Backend: каталог опубликованных карточек — фильтр категория/подкатегория, поиск (название+провайдер), сортировка популярные/новые, пагинация, агрегация счётчиков дерева
- [ ] 6.2 Frontend: каталог — сетка карточек (обложка, tagline, провайдеры «+N», запуски), пагинация, сортировка, поиск с пустым состоянием
- [ ] 6.3 Frontend: дерево категорий с счётчиками, раскрытие, хлебные крошки
- [ ] 6.4 Frontend: страница карточки — вкладки «Обзор» (rich-описание, метрики), «Сервисы и endpoints»
- [ ] 6.5 Landing-шапка маркетплейса (hero с счётчиком use-case)

## 7. Milestone 6 — Исполнение сценариев (`scenario-execution`)

- [ ] 7.1 Backend: модель Run + движок — последовательное исполнение, резолв маппингов и `{{token}}`-шаблонов, HTTP-вызовы endpoints с таймаутами и лимитами
- [ ] 7.2 Backend: SSE-стрим событий запуска (step:start/done, page:required, run:done/error), журнал событий в Run, reconnect по runId
- [ ] 7.3 Backend: шаги delay (пауза), periodic (поллинг с прогрессом до завершения), вложенный сценарий; остановка при ошибке с диагностикой шага
- [ ] 7.4 Backend: пауза `waiting_input` на страницах шагов, продолжение POST'ом с данными формы; инкремент счётчика запусков
- [ ] 7.5 Frontend: простой режим — форма входов первого шага, прогресс шагов, итоговая сводка (заголовок, вердикт, поля), «Запустить снова»
- [ ] 7.6 Frontend: playground — пошаговый запуск, JSON-ответы с таймингами, «выполнить все», сброс, финальный статус с суммарным временем
- [ ] 7.7 Frontend: рендер страниц шагов в запуске (fields с required-валидацией, text), продолжение потока
- [ ] 7.8 Интеграционные тесты движка: маппинги, ошибки API, циклы, поллинг

## 8. Milestone 7 — Файлы с докачкой

- [ ] 8.1 Backend: chunked upload (init/part/complete/abort) в MinIO, состояние частей в Mongo, валидация формата и лимита из страницы шага
- [ ] 8.2 Frontend: дропзона — прогресс (проценты/байты), пауза/продолжение, отмена
- [ ] 8.3 Frontend/Backend: восстановление после обрыва — сохранённые чанки, «Возобновить загрузку» с места обрыва
- [ ] 8.4 Backend: передача файла в endpoint провайдера (multipart целиком / чанки octet-stream), стадия обработки через поллинг статуса, результат (поля/текст)
- [ ] 8.5 Frontend: экран результата файлового шага (извлечённые поля, «Загрузить другой файл»)

## 9. Milestone 8 — Стабилизация и запуск

- [ ] 9.1 Сквозной E2E-сценарий: импорт API → сборка мультисервисного сценария со страницами → публикация → запуск из маркетплейса (Playwright)
- [ ] 9.2 Демо-данные (seed): 3-4 приложения и 6+ карточек по мотивам прототипа
- [ ] 9.3 Аудит безопасности: SSRF, шифрование констант-секретов, лимиты запусков, rate limiting auth
- [ ] 9.4 UX-полировка: тосты, пустые состояния, скелетоны, адаптив основных экранов
- [ ] 9.5 Документация: README (запуск, архитектура), обновление openspec/config.yaml (context проекта)
