# scenario-execution

## ADDED Requirements

### Requirement: Движок исполнения сценария
Система SHALL исполнять шаги сценария последовательно на стороне backend: для каждого шага разрешать входы по маппингу (выходы предыдущих шагов, ввод пользователя, константы, включая подстановку `{{token}}`-шаблонов), выполнять HTTP-вызов endpoint стороннего API и сохранять ответ как выходы шага. Шаг «Задержка» MUST приостанавливать поток на заданное время; «Периодический запрос» MUST опрашивать endpoint с интервалом до признака завершения, публикуя прогресс; «Другой сценарий» MUST исполнять вложенный сценарий как один шаг. Ошибка шага MUST останавливать запуск с диагностикой (шаг, endpoint, статус ответа).

#### Scenario: Успешный запуск потока
- **WHEN** пользователь запускает сценарий из N шагов с валидными входами
- **THEN** система выполняет все шаги по порядку, передавая данные по маппингу, и возвращает итоговый результат с суммарным временем

#### Scenario: Ошибка стороннего API
- **WHEN** endpoint шага возвращает ошибку
- **THEN** запуск останавливается, пользователь видит на каком шаге и какая ошибка произошла

### Requirement: Простой режим запуска
Вкладка «Запуск» SHALL по умолчанию предлагать простой режим: форма с входами первого шага, кнопка «Получить результат», индикация выполняемых шагов (название, сервис, статус), и итоговая сводка результата (заголовок, оценка/вердикт при наличии, ключевые поля) с кнопкой «Запустить снова». Счётчик запусков карточки MUST инкрементироваться при каждом завершённом запуске.

#### Scenario: Запуск в один клик
- **WHEN** пользователь вводит значение (например, ИНН) и нажимает «Получить результат»
- **THEN** система показывает прогресс по шагам и итоговую сводку без отображения технических деталей

### Requirement: Режим playground
Система SHALL предоставлять пошаговый режим: каждый шаг показывает метод, путь, сервис, описание и поле ввода; пользователь запускает шаги по одному и видит JSON-ответ каждого шага с временем выполнения; MUST быть кнопки «Сбросить» и запуск всех шагов подряд с финальным статусом «Сценарий выполнен» и суммарным временем.

#### Scenario: Пошаговое выполнение
- **WHEN** пользователь выполняет шаги по одному в playground
- **THEN** после каждого шага отображается его ответ (JSON, тайминг, сервис), а после последнего — итоговый статус

### Requirement: Загрузка файлов с докачкой
Для файловых шагов система SHALL загружать файл чанками с отображением прогресса (проценты и байты), поддерживать паузу/продолжение и отмену. При обрыве соединения загруженные чанки MUST сохраняться, и пользователь MAY возобновить загрузку с места обрыва. После загрузки система SHALL показывать стадию обработки (опрос статуса) и результат (извлечённые поля/текст) с возможностью загрузить другой файл. Файлы MUST проверяться на допустимый формат и лимит размера, заданные страницей шага.

#### Scenario: Возобновление после обрыва
- **WHEN** соединение обрывается на середине загрузки
- **THEN** система показывает «Соединение прервалось на X% — загруженные чанки сохранены» и после «Возобновить загрузку» продолжает с того же места

### Requirement: Отображение страниц шагов при запуске
При исполнении сценария система SHALL показывать привязанные к шагам страницы: «Ввод полей» запрашивает данные (обязательные поля блокируют продолжение), «Загрузка файла» принимает файл, «Отображение текста» показывает контент; введённые данные подставляются во входы шага согласно маппингу.

#### Scenario: Страница ввода посреди потока
- **WHEN** исполнение доходит до шага со страницей «Ввод полей»
- **THEN** поток приостанавливается, пользователь заполняет форму, и после отправки исполнение продолжается с этими данными
