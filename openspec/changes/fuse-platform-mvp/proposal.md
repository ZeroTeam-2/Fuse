# Fuse Platform MVP

## Why

Fuse — платформа, которая превращает подключённые API в готовые «карточки-сценарии»: владелец API импортирует OpenAPI-спецификацию, собирает из endpoints многошаговый сценарий (в том числе из нескольких API сразу), а обычный пользователь запускает его из маркетплейса в один клик — без кода и чтения документации. Дизайн-прототип (Fuse.html) полностью описывает продукт; репозиторий сейчас — пустой каркас монорепо (NuxtJS + NestJS + MongoDB). Нужно зафиксировать функциональные требования по SDD и построить план реализации MVP по milestone.

## What Changes

- Аутентификация и профиль пользователя: регистрация, вход, смена пароля, аватар, личные данные, сводка «мои API / мои сценарии».
- Управление приложениями (API-провайдерами): создание приложения, импорт OpenAPI (JSON по ссылке) с автоматическим разбором endpoints, просмотр endpoints, обновление спецификации с diff «новые endpoints», публикация/скрытие.
- Конструктор сценариев: карточка сценария (название, обложка, rich-описание, категория), редактор шагов с типами шагов — вызов endpoint, вложенный сценарий, задержка, загрузка файла (одним запросом или чанками S3-multipart, с опциональным опросом статуса), периодический запрос; маппинг входных данных шага (выход предыдущих шагов / ввод пользователя / константа) с авто-подстановкой; перестановка шагов drag-and-drop; публикация в маркетплейс.
- Пользовательские страницы шагов: к любому шагу привязывается страница «что увидит пользователь» трёх типов — ввод полей, загрузка файла, отображение текста; редактор страницы с live-превью.
- Маркетплейс: каталог карточек с деревом категорий/подкатегорий и счётчиками, поиск, сортировка (популярные/новые), пагинация; страница карточки — обзор, задействованные API и endpoints, переход к запуску.
- Исполнение сценариев (runtime): простой запуск «ввёл данные → получил результат», пошаговый playground с логом запросов/ответов и таймингами, отзывчивый прогресс; загрузка файлов чанками с паузой/возобновлением и восстановлением после обрыва соединения; отображение результата (поля, текст, транскрипт по спикерам).

## Capabilities

### New Capabilities

- `auth-profile`: регистрация, вход, сессии, профиль (ФИО, email, аватар), смена пароля, сводка опубликованного.
- `api-app-management`: CRUD приложений, импорт и обновление OpenAPI-спецификаций, реестр endpoints, публикация приложения.
- `scenario-builder`: CRUD сценариев, редактор шагов всех типов, маппинг данных между шагами, категории, публикация карточек.
- `step-pages`: конструктор пользовательских страниц шагов (fields / file / text) с превью.
- `marketplace`: каталог, категории, поиск, сортировка, пагинация, страница карточки.
- `scenario-execution`: движок исполнения сценариев, простой и пошаговый режимы запуска, chunked-загрузка файлов с resume, поллинг статуса, представление результатов.

### Modified Capabilities

_нет — это первый change, существующих спецификаций нет._

## Impact

- `apps/backend` (NestJS + MongoDB): модули auth, users, apps, scenarios, marketplace, execution; хранение OpenAPI-спецификаций и схем endpoints; прокси-исполнение HTTP-запросов к сторонним API; chunked upload.
- `apps/frontend` (Nuxt): страницы маркетплейса, карточки, запуска, «Мои API», «Мои сценарии», редакторы приложения/сценария/страницы шага, профиль.
- `packages/shared`: общие типы (App, Endpoint, Scenario, Step, Page, Run), enum'ы типов шагов и страниц.
- Инфраструктура: docker-compose (MongoDB), генерация типов из OpenAPI backend'а (`gen:types` уже настроен).
- Внешние зависимости: доступ к сторонним API по их OpenAPI; хранилище файлов для чанковой загрузки (S3-совместимое).
