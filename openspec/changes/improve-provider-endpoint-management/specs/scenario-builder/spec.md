## ADDED Requirements

### Requirement: Вкладка «Окружения» сценария
Редактор сценария SHALL иметь отдельную вкладку **«Окружения»**. Вкладка MUST перечислять всех поставщиков (apps), задействованных в шагах сценария, и для каждого позволять выбрать используемое окружение из окружений этого поставщика. По умолчанию для каждого поставщика MUST быть выбран Prod. Выбор MUST сохраняться в составе сценария как соответствие «поставщик → окружение».

Список поставщиков на вкладке MUST быть производным от шагов: поставщик появляется, когда в сценарий добавлен шаг по его endpoint, и исчезает, когда таких шагов не остаётся. Исполнение шага MUST использовать окружение, выбранное для поставщика этого шага (см. capability `api-app-environments`).

#### Scenario: Выбор окружения для поставщика
- **WHEN** в сценарии есть шаги по поставщику, и пользователь на вкладке «Окружения» выбирает для него «Staging»
- **THEN** выбор сохраняется, и последующее исполнение шагов этого поставщика идёт по `baseUrl` окружения Staging

#### Scenario: Список поставщиков выводится из шагов
- **WHEN** пользователь добавляет первый шаг по новому поставщику
- **THEN** на вкладке «Окружения» появляется этот поставщик с выбором окружения по умолчанию Prod

#### Scenario: По умолчанию Prod
- **WHEN** пользователь не менял окружение поставщика
- **THEN** для него используется окружение Prod

## MODIFIED Requirements

### Requirement: Типы шагов
Редактор шагов SHALL поддерживать пять типов шага:
1. **Endpoint API** — вызов метода выбранного приложения (выбор приложения с поиском → выбор endpoint);
2. **Другой сценарий** — вложение готового сценария (его входы → выходы); система MUST предотвращать циклические вложения;
3. **Задержка** — пауза на заданное число секунд перед следующим шагом (пресеты 1/3/5/10 сек и произвольное значение);
  4. **Файл** — загрузка файла пользователем; система автоматически выбирает режим по размеру: ≤ 10 МБ (настраиваемый порог) — single-upload `multipart/form-data`, > 10 МБ — мультичанковая загрузка S3 `application/octet-stream` с паузой/возобновлением; Content-Type MUST определяться режимом; опционально — endpoint опроса статуса обработки с интервалом и полем прогресса (в байтах);
5. **Периодический запрос** — опрос выбранного endpoint с интервалом (1–600 сек) и полем прогресса из схемы ответа.

При выборе endpoint (типы «Endpoint API» и «Периодический запрос») список endpoints выбранного приложения MUST выводиться **сворачиваемыми блоками по тегам** (endpoints без тегов — в блоке «Прочее») и MUST сопровождаться **сквозным поиском** по методу, пути, summary и имени блока/тега. Стиль поиска и группировки MUST быть единым с карточкой приложения. При активном поиске отображаются только совпавшие endpoints; блоки с совпадениями раскрываются.

#### Scenario: Добавление API-шага
- **WHEN** пользователь выбирает тип «Endpoint API», приложение и endpoint, задаёт название шага и нажимает «Добавить шаг»
- **THEN** шаг появляется в потоке с методом, путём и названием приложения

#### Scenario: Выбор endpoint сгруппирован и с поиском
- **WHEN** пользователь выбрал приложение и открыл выбор endpoint
- **THEN** endpoints показаны сворачиваемыми блоками по тегам с полем сквозного поиска в едином стиле с карточкой приложения; ввод запроса оставляет только совпавшие endpoints

#### Scenario: Вложение сценария
- **WHEN** пользователь выбирает тип «Другой сценарий» и указывает сценарий
- **THEN** шаг принимает входы вложенного сценария и отдаёт его выходы; сценарии, приводящие к циклу, не предлагаются

#### Scenario: Настройка файлового шага
- **WHEN** пользователь настраивает файловый шаг и включает опрос статуса
- **THEN** система требует выбрать endpoint статуса, интервал и поле прогресса; режим single/chunked определяется автоматически по размеру файла при загрузке
