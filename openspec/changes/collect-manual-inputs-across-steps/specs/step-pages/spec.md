## ADDED Requirements

### Requirement: Привязка поля страницы к значению шага

Поле страницы «Ввод полей» SHALL иметь явную привязку к значению того шага, к которому привязана страница: к входному параметру (`inn`) либо к операнду условия фильтрации (`filter:status`). Редактор страницы MUST предлагать выбор из ручных значений **этого** шага и MUST NOT требовать, чтобы подпись поля совпадала с техническим ключом параметра: подпись остаётся свободной («ИНН организации»), а связь задаётся привязкой.

Поле **без** привязки MUST сохранять прежнее поведение — его значение уходит во входы шага под собственным ключом поля, и оно считается покрывающим параметр, только если ключ поля совпал с ключом параметра. Существующие сценарии, где подпись поля была названа ключом параметра ради этого совпадения, MUST продолжать работать без правок.

При исполнении данные страницы MUST переводиться из ключей полей в ключи значений шага согласно привязке — до того, как попасть в резолвер маппингов.

#### Scenario: Поле связывается с параметром шага

- **WHEN** редактор добавляет на страницу шага поле с подписью «ИНН организации» и указывает, что оно заполняет параметр `inn`
- **THEN** введённое пользователем значение попадает во вход `inn` этого шага, а подпись поля остаётся человекочитаемой

#### Scenario: Поле связывается со значением условия фильтрации

- **WHEN** у шага есть условие фильтрации со значением в режиме ручного ввода, и редактор указывает, что поле страницы заполняет это значение
- **THEN** введённое значение применяется как операнд условия отбора элемента массива, и это значение не спрашивается общей формой перед запуском

#### Scenario: Поле без привязки работает по-старому

- **WHEN** у поля страницы привязка не задана, а его ключ совпадает с ключом параметра шага
- **THEN** значение по-прежнему уходит в этот параметр, и сценарий, настроенный до появления привязок, продолжает работать

### Requirement: Страница ввода как источник ручных значений своего шага

Страница ввода полей SHALL считаться источником ручных значений **того шага**, к которому она привязана. Значение шага MUST считаться покрытым страницей, если поле страницы привязано к нему (либо, при отсутствии привязки, если ключ поля совпадает с ключом параметра). Покрытые значения MUST NOT попадать в общую форму ручных входов перед запуском — пользователь MUST NOT спрашиваться о них дважды.

Ручные значения шага, **не** покрытые его страницей, MUST собираться общей формой перед запуском наравне со значениями шагов без страницы.

Данные, введённые на странице шага, MUST иметь приоритет над значением того же входа, пришедшим из входов запуска.

#### Scenario: Значение покрыто страницей и не спрашивается дважды

- **WHEN** параметр `inn` шага помечен ручным вводом, а на странице этого шага есть поле, привязанное к `inn`
- **THEN** значение спрашивается только страницей шага при исполнении, и в общей форме перед запуском поле `inn` этого шага не показывается

#### Scenario: Страница покрывает не все ручные значения шага

- **WHEN** у шага два параметра с ручным вводом, а страница шага содержит поле только для одного из них
- **THEN** покрытый параметр спрашивает страница, непокрытый — общая форма перед запуском, и оба доезжают до входов шага

#### Scenario: Данные страницы перекрывают входы запуска

- **WHEN** значение того же входа пришло и из входов запуска, и со страницы шага
- **THEN** worker использует значение, введённое на странице
