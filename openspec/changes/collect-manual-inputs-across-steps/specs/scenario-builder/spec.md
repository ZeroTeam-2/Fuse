## MODIFIED Requirements

### Requirement: Маппинг данных между шагами
Для каждого входного поля шага система SHALL позволять выбрать источник: выход одного из предыдущих шагов, ввод пользователя или константа. Система MUST автоматически подставлять источник по совпадению ключа или метки с выходами предыдущих шагов. Заголовки авторизации MUST по умолчанию получать константу `Bearer {{access_token}}`. Панель шага SHALL показывать выходные поля endpoint с примерами значений. Шаг без входных данных MAY стоять первым в потоке.

Для значений, помеченных ручным вводом, панель шага MUST честно показывать, **откуда** значение придёт при запуске: со страницы ввода этого шага (если поле страницы привязано к этому значению) либо из общей формы ручных входов перед запуском. Панель MUST NOT обещать механизм сбора, которого нет: подсказка о запросе значения при запуске MUST соответствовать фактическому источнику.

#### Scenario: Автоподстановка источника
- **WHEN** пользователь добавляет шаг, у которого поле `id` совпадает с выходом предыдущего шага
- **THEN** система автоматически маппит поле на выход этого шага, оставляя возможность сменить источник вручную

#### Scenario: Источник ручного значения показан в панели шага
- **WHEN** у шага есть параметр с ручным вводом, а на странице этого шага нет поля, привязанного к нему
- **THEN** панель шага показывает, что значение будет запрошено общей формой перед запуском, а значение, к которому поле страницы привязано, помечено как запрашиваемое страницей шага
