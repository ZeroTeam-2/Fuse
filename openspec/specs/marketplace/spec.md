# marketplace

## Purpose

Публичный каталог опубликованных сценариев с деревом категорий, поиском и детальными карточками сценариев.

## Requirements

### Requirement: Каталог сценариев
Маркетплейс SHALL показывать опубликованные карточки сценариев: обложка, название, tagline, провайдер(ы) API (с меткой «+N» при нескольких), число запусков. Каталог MUST поддерживать пагинацию с меткой диапазона и сортировку «Популярные» / «Новые». Неопубликованные карточки MUST NOT попадать в выдачу.

#### Scenario: Просмотр каталога
- **WHEN** пользователь открывает маркетплейс
- **THEN** система показывает страницу опубликованных карточек, отсортированных по популярности, и общий счётчик

### Requirement: Дерево категорий
Каталог SHALL иметь боковое дерево категорий с подкатегориями, счётчиками карточек и пунктом «Все сценарии»; категории раскрываются/сворачиваются, выбор подкатегории фильтрует выдачу и отражается в хлебных крошках. Справочник категорий MUST быть расширяемым (Данные, Документы, Аудио, Видео, Изображения, Текст и NLP, Гео, Финансы и др.).

#### Scenario: Фильтрация по подкатегории
- **WHEN** пользователь выбирает подкатегорию «Распознавание» в категории «Документы»
- **THEN** выдача содержит только карточки этой подкатегории, счётчики и крошки обновляются

### Requirement: Поиск
Маркетплейс SHALL поддерживать текстовый поиск по названию карточки и имени провайдера. Поиск MUST обновлять счётчики дерева категорий и показывать пустое состояние «По вашему запросу ничего не найдено».

#### Scenario: Пустой результат
- **WHEN** запрос не совпадает ни с одной карточкой
- **THEN** система показывает пустое состояние и нулевые счётчики, сбрасываемые очисткой запроса

### Requirement: Страница карточки
Карточка сценария SHALL иметь вкладки: «Обзор» (описание в rich-формате, число запусков, число поставщиков API, число endpoints, категория, кнопка «Перейти к запуску»), «Запуск» и «Сервисы и endpoints» (список поставщиков API и задействованных endpoints с методами и путями).

Страница карточки SHALL быть публичной: гость видит её целиком, включая вкладку «Запуск» с превью шагов сценария. Публичное представление опубликованного сценария MUST включать его шаги. Сам запуск MUST требовать аутентификации (см. capability `auth-profile`).

#### Scenario: Просмотр обзора
- **WHEN** пользователь открывает карточку из каталога
- **THEN** система показывает вкладку «Обзор» с отрендеренным описанием и метриками карточки

#### Scenario: Гость смотрит вкладку «Запуск»
- **WHEN** неаутентифицированный пользователь открывает вкладку «Запуск» карточки
- **THEN** система показывает панель запуска с превью шагов сценария (не сообщение об ошибке доступа), но запуск остаётся недоступным до входа
