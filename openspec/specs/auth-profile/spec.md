# auth-profile

## Purpose

Аутентификация пользователей через Yandex OAuth, управление личным профилем и сводка пользовательских публикаций (API и сценарии).

## Requirements

### Requirement: Вход через Яндекс (Yandex OAuth)
Система SHALL предоставлять вход через Yandex OAuth (Yandex ID) без отдельной регистрации по email. При первом входе система автоматически создаёт пользователя на основе профиля Яндекса (ФИО, email, аватар). Все разделы, кроме публичного маркетплейса, MUST быть доступны только аутентифицированному пользователю.

#### Scenario: Успешный вход через Яндекс
- **WHEN** пользователь нажимает «Войти через Яндекс» и подтверждает доступ в Яндекс ID
- **THEN** система создаёт сессию (JWT в httpOnly cookie), при необходимости создаёт пользователя, и открывает маркетплейс с данными пользователя (инициалы в шапке)

#### Scenario: Отказ от авторизации
- **WHEN** пользователь отменяет доступ в Яндекс ID
- **THEN** система остаётся на странице входа и не создаёт сессию

#### Scenario: Недействительный OAuth-код
- **WHEN** callback содержит недействительный или истекший код
- **THEN** система показывает ошибку авторизации и предлагает повторить вход

### Requirement: Профиль пользователя
Система SHALL хранить и позволять редактировать личные данные: ФИО, email, фото профиля (загрузка и удаление; при первом входе подставляется аватар из профиля Яндекса). Личные данные MUST быть видны только самому пользователю и владельцам API, чьи сценарии он запускает.

#### Scenario: Изменение личных данных
- **WHEN** пользователь меняет ФИО или email и нажимает «Сохранить изменения»
- **THEN** система сохраняет данные и подтверждает сохранение

#### Scenario: Загрузка аватара
- **WHEN** пользователь загружает файл изображения
- **THEN** система сохраняет аватар и отображает его вместо инициалов; не-изображения отклоняются

### Requirement: Сводка публикаций в профиле
Профиль SHALL показывать списки «Мои API» (с количеством endpoints, хостом, версией и статусом публикации) и «Мои сценарии» (со статусом, числом шагов и tagline) с пагинацией и переходами к управлению.

#### Scenario: Просмотр сводки
- **WHEN** пользователь открывает профиль
- **THEN** система показывает счётчики и постраничные списки его API и сценариев со статусами публикации
