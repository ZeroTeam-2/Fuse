## Requirements
### Requirement: Окружения приложения
Приложение SHALL иметь набор окружений (Environments). Каждое приложение MUST при создании получать окружение по умолчанию **Prod**, которое MUST NOT удаляться. Каждое окружение MUST иметь имя (уникальное в пределах приложения) и **набор переменных**. Набор переменных MUST содержать как минимум базовую переменную **`baseUrl`** — абсолютный адрес со схемой и хостом, к которому присоединяется путь endpoint'а при исполнении. Модель набора переменных MUST быть расширяемой (в текущей итерации единственный элемент — `baseUrl`). Переменная `baseUrl` окружения Prod MUST инициализироваться значением `baseUrl`, выведенным при импорте OpenAPI.

Пользователь SHALL иметь возможность создавать новые окружения и удалять любые, кроме Prod. Удаление окружения MUST NOT удалять само приложение и его endpoints.

#### Scenario: Окружение Prod по умолчанию
- **WHEN** приложение создаётся импортом OpenAPI
- **THEN** у приложения есть окружение Prod с набором переменных, где `baseUrl` равен выведенному при импорте `baseUrl`

#### Scenario: Создание нового окружения
- **WHEN** владелец на карточке провайдера добавляет окружение (например, «Staging») и задаёт его переменную `baseUrl`
- **THEN** система сохраняет новое окружение с указанным набором переменных, не затрагивая Prod

#### Scenario: Удаление окружения
- **WHEN** владелец удаляет окружение, отличное от Prod
- **THEN** окружение исчезает, приложение и его endpoints сохраняются

#### Scenario: Prod нельзя удалить
- **WHEN** владелец пытается удалить окружение Prod
- **THEN** система запрещает удаление и объясняет, что Prod обязателен

### Requirement: Управление переменными окружения
Система SHALL позволять редактировать переменные каждого окружения. Значение переменной `baseUrl` MUST быть валидным абсолютным URL со схемой (`http`/`https`) и хостом; пустое или невалидное значение MUST отклоняться понятной ошибкой. Переменную `baseUrl` MUST NOT быть возможности удалить из набора.

#### Scenario: Изменение baseUrl
- **WHEN** владелец меняет переменную `baseUrl` окружения на валидный абсолютный адрес
- **THEN** система сохраняет новое значение

#### Scenario: Невалидный baseUrl
- **WHEN** владелец вводит значение `baseUrl` без схемы или хоста
- **THEN** система отклоняет сохранение с понятной ошибкой и не меняет прежнее значение

### Requirement: Исполнение относительно окружения
Исполнение шага сценария по endpoint'у приложения SHALL резолвить путь endpoint'а относительно переменной `baseUrl` окружения, **выбранного для поставщика этого шага в сценарии** (см. capability `scenario-builder`). При отсутствии явного выбора система MUST использовать окружение Prod. Если окружение или его `baseUrl` недоступны, система MUST падать на `App.baseUrl`. Полученный абсолютный URL MUST быть вызываемым, как и прежде при едином `baseUrl`.

#### Scenario: Исполнение использует выбранное окружение
- **WHEN** для поставщика шага в сценарии выбрано окружение «Staging»
- **THEN** путь endpoint'а присоединяется к `baseUrl` окружения Staging, и вызов идёт по этому абсолютному URL

#### Scenario: Исполнение по умолчанию использует Prod
- **WHEN** шаг вызывает endpoint приложения, для поставщика которого окружение в сценарии не выбрано
- **THEN** путь присоединяется к `baseUrl` окружения Prod, и вызов идёт по этому абсолютному URL

#### Scenario: Обратная совместимость существующих приложений
- **WHEN** открывается приложение, созданное до появления окружений (есть только `baseUrl`)
- **THEN** система показывает окружение Prod с переменной `baseUrl`, равной прежнему `baseUrl`, и исполнение работает без изменений
