## Requirements
### Requirement: Smoke-тест локальной инфраструктуры

Проект SHALL предоставлять автоматический тест, который проверяет, что поднятая локальная инфраструктура пригодна к работе: очередь и DLQ существуют, бакет существует, запись и чтение проходят. Тест MUST падать с внятным сообщением, если инфраструктура не поднята.

#### Scenario: Инфраструктура поднята

- **WHEN** после `pnpm infra` выполняется smoke-тест инфраструктуры
- **THEN** тест подтверждает наличие очереди `scenario-execution` и её DLQ
- **AND** подтверждает наличие бакета и успешный round-trip: отправленное в очередь сообщение читается обратно, а загруженный объект скачивается из S3

#### Scenario: Инфраструктура не поднята

- **WHEN** smoke-тест запускается при остановленной инфраструктуре
- **THEN** тест падает с сообщением, из которого понятно, что нужно выполнить `pnpm infra`
- **AND** не зависает в ожидании подключения

### Requirement: E2E-тесты UI поверх сид-данных

Проект SHALL предоставлять e2e-тесты, которые прогоняют ключевые пользовательские сценарии в браузере от имени сид-пользователя. Тесты MUST выполняться командой из `package.json` и MUST использовать вход по подписанной куке, а не Yandex OAuth.

#### Scenario: Запуск сценария доходит до терминального статуса

- **WHEN** e2e-тест входит под сид-пользователем и запускает сид-сценарий из UI
- **THEN** UI показывает переход запуска в терминальный статус
- **AND** тест не проходит, если статус остаётся в «Выполняем сценарий…» дольше отведённого таймаута

#### Scenario: Создание сценария не плодит дубликаты

- **WHEN** e2e-тест создаёт сценарий через форму создания
- **THEN** создаётся ровно один сценарий
- **AND** тест падает, если в списке появляется более одного сценария с этим названием

#### Scenario: Ошибка навигации не выдаётся за ошибку создания

- **WHEN** переход в редактор после успешного создания сценария не удаётся
- **THEN** пользователю не показывается сообщение «Не удалось создать сценарий»
- **AND** повторное нажатие не создаёт второй сценарий
