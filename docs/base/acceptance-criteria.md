# Критерии приёмки инициализации монорепозитория

Документ описывает минимально необходимые проверки, подтверждающие корректную инициализацию проекта. Каждый пункт должен проходить на чистой машине после `git clone`.

---

## 1. Установка зависимостей

```bash
pnpm install
```

| #   | Проверка                                                           | Ожидаемый результат                                      |
| --- | ------------------------------------------------------------------ | -------------------------------------------------------- |
| 1.1 | Команда завершается без ошибок                                     | exit code 0                                              |
| 1.2 | Папка `node_modules` создана в корне и во всех packages/apps       | файлы присутствуют                                       |
| 1.3 | Symlink `@fuse/shared` разрешён в `apps/frontend` и `apps/backend` | `node_modules/@fuse/shared` → `../../../packages/shared` |
| 1.4 | Git hooks активированы                                             | `git config core.hooksPath` → `.githooks`                |

---

## 2. Сборка пакетов

```bash
pnpm build
```

| #   | Проверка                                        | Ожидаемый результат                         |
| --- | ----------------------------------------------- | ------------------------------------------- |
| 2.1 | `packages/shared` собирается без ошибок         | папка `dist/` появилась в `packages/shared` |
| 2.2 | `apps/backend` собирается без ошибок TypeScript | папка `dist/` в `apps/backend`              |
| 2.3 | `apps/frontend` собирается без ошибок           | папка `.output/` в `apps/frontend`          |

## 4. Docker / MongoDB

```bash
docker compose up -d
docker compose ps
```

| #   | Проверка                        | Ожидаемый результат                                                                   |
| --- | ------------------------------- | ------------------------------------------------------------------------------------- |
| 4.1 | Контейнер `mongo` запустился    | статус `running`                                                                      |
| 4.2 | MongoDB доступна на порту 27017 | `docker compose exec mongo mongosh --eval "db.runCommand({ ping: 1 })"` → `{ ok: 1 }` |
| 4.3 | Volume создан                   | `docker volume ls` показывает `fuse_mongo_data`                                       |

## 5. Бэкенд (NestJS)

```bash
cp .env.example .env   # заполнить MONGODB_URL
cd apps/backend
pnpm dev
```

| #   | Проверка                       | Ожидаемый результат                                                                    |
| --- | ------------------------------ | -------------------------------------------------------------------------------------- |
| 5.1 | Приложение стартует без ошибок | в логах `Application is running on: http://localhost:3001`                             |
| 5.2 | Валидация .env работает        | удалить `MONGODB_URL` из `.env` → старт падает с ошибкой zod                           |
| 5.3 | MongoDB подключена             | нет ошибок `MongooseError` в логах                                                     |
| 5.4 | Scalar UI доступен             | `GET http://localhost:3001/api/docs` → 200, открывается Scalar                         |
| 5.5 | OpenAPI схема доступна         | `GET http://localhost:3001/api/schema.json` → валидный JSON с полем `openapi: "3.0.0"` |
| 5.6 | Health-check эндпоинт          | `GET http://localhost:3001/` → 200                                                     |

## Итоговый чеклист (быстрая проверка)

```bash
# 1. Установка
pnpm install

# 2. Docker
docker compose up -d && docker compose ps

# 3. Бэк
cd apps/backend && pnpm dev &

# 4. Генерация типов
cd ../.. && pnpm gen:types

# 5. Фронт
cd apps/frontend && pnpm dev &

# 6. Сборка
cd ../.. && pnpm build

# 7. Линт
pnpm lint
```

Все команды должны завершиться с exit code `0`. Если хотя бы одна упала — инициализация считается неуспешной.
